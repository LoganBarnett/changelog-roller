A simple ~CHANGELOG~ roller.  Moves ~Upcoming~ to your newly released version,
and creates a new ~Upcoming~ section.

Currently only supports ~org-mode~ files, but could support Markdown in the
future.

* Introduction

~changelog-roller~ is a simple tool that provides a staging area for upcoming
changes (defaults to ~Upcoming~ as a heading).  As changes roll in, authors of
the change log add meaningful changes to the ~Upcoming~ heading.  When the
release time is right, invoke ~changelog-roller~ with a new version.  This will
effectively change the ~Upcoming~ section to be that new version, and emit a
new, blank ~Upcoming~ section to stage new changes upon.

* Example

Assume a ~CHANGELOG.org~ document like this:

#+begin_src org :results none :exports code
,* changelog
,** Upcoming
,*** Breaking
1. Removed important feature ~shiny~!
,*** Additions
,*** Fixes
,** v0.1.0
,*** Breaking
,*** Additions
1. Lots of new features!
,*** Fixes
#+end_src

Then run ~changelog-roller roll --version v0.2.0~ (which reads
~CHANGELOG.org~ by default) and observe the new document:

#+begin_src org :results none :exports code
,* changelog
,** Upcoming
,*** Breaking
,*** Additions
,*** Fixes
,** v0.2.0
,*** Breaking
1. Removed important feature ~shiny~!
,*** Additions
,*** Fixes
,** v0.1.0
,*** Breaking
,*** Additions
1. Lots of new features!
,*** Fixes
#+end_src

* Adding entries

To add a new entry to the upcoming section without hand-editing the file,
use the ~insert-item~ subcommand:

#+begin_src sh :results none :exports code
changelog-roller --in-place insert-item \
  --heading Additions \
  --body "Added shiny new thing"
#+end_src

If the named heading already exists under ~Upcoming~, the new entry is
appended to its numbered list as the next item.  If it does not exist, the
heading is created and the entry becomes its first list item.  Headings
are matched by exact string, so ~Fix~ and ~Fixes~ are different headings.

* Subcommands

- ~roll --version <V>~ — stamp ~Upcoming~ as version ~V~ and start a fresh
  upcoming section.
- ~insert-item --heading <H> --body <B>~ — add an entry under heading ~H~
  in the upcoming section.
- ~ready-to-roll~ — exit non-zero if the upcoming section has no content;
  useful as a CI gate.
- ~check-additions --base <REF>~ — exit non-zero if no new entries have
  been added to the upcoming section relative to a git ref.

Without ~--in-place~, ~roll~ and ~insert-item~ write the updated changelog
to stdout.
